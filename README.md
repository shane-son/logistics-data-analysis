# logistics-data-analysis

## 프로젝트 개요
물류센터 작업 효율 개선을 위한 데이터 분석 프로젝트입니다.

## 사용 기술
- Python
- Pandas
- SQL

## 1. 문제정의
 운영 과정에서 취소 상품이 발생할 경우 전산 처리 후 상품을 재사용 할 수 있지만(단, 상품에 파손이 없는 가공 상품에 한한다.) 처리 지연 및 관리 비효율로 인해 재사용 가능한 재고가 적시에 활용되지 못하는 문제가 발생한다. 이로 인해 공정 미집,오집 등의 사유로 추가 취소건이 발생할 가능성이 존재한다.

## 2. 가설
- 취소 상품 전산처리 자동화를 통해 재사용 가능한 재고 데이터를 생성하여 취소 재고의 재사용률 분석
- 오집/미집 과발생 구간 식별
- 추가 취소 감소를 위한 운영 개선 방안 도출
- 데이터를 이용한 모니터링 및 분석

## 3. 업무 흐름 개선

### 3-1. 개선 전 업무 흐름
-주문 발생 → 취소 발생 → 수동 전산 처리 → 재사용시 상품 직접 실물 탐색

### 3-2. 개선 후 업무 흐름
-주문 발생 → 취소 발생 → 자동 전산 처리 → 재고 전산 즉시 반영 → 전산기반 재사용 상품 탐색

### 3-3. 기대효과
- 상품 탐색 시간 단축
- 재사용 상품 탐색 누락 확률 감소
- 상품 탐색 작업자 의존도 감소

## 4.핵심 분석 질문
- 취소 재고의 재사용률은 어느 정도인가?
- 오집/미집 중 어떤 사유에서의 재사용 실패가 많은가?(이거는 어차피 반반일건데 미집은 어차피 없어서 불가하니까 오집에 비중을 두는게 맞을거 같기도 하고)
- 처리 지연 시간과 재사용률의 관계는 있는가?
- 재사용 실패가 추가 취소로 이어지는 비율은 어느 정도인가?
- 특정 상품군에서 과발생 패턴이 존재하는가?

## 5. 시뮬레이션 설계
- 주문 데이터 생성 → 일정 비율 취소 발생 → 취소 사유 부여(오집 / 미집 / 집품파손 / 리빈파손) → 전산 처리 시간 생성 → 재고 반영 여부 결정 → 재사용 여부 결정 → 추가 취소 여부 생성

### 5-1. 취소 사유 부여 및 시뮬레이션 가정

취소 발생 데이터에 대해 작업자 숙련도와 작업 환경에 따른 오류 가능성을 반영하여 취소 사유를 설정하였다.  
오류 발생 가능성은 작업자 숙련도, 주문 피크 시간, 오진열 발생 여부를 기준으로 가정하였다.

예시:

- 집품 작업자 비율
  - 숙련자 : 67%
  - 비숙련자 : 33% (신규 20명 / 전체 60명 기준)

- 작업자 숙련도에 따른 기본 오류 발생률
  - 숙련자 평균 HTP : 195
  - 숙련자 오류 발생률 : 약 1% (2 / 195)
  - 비숙련자 평균 HTP : 160
  - 비숙련자 오류 발생률 : 약 6.25% (10 / 160)

- 특이 상황에 따른 추가 오류 발생률

  - 주문 피크 시간  
    피크 시간대의 작업 환경 변동성을 반영하여 추가 오류 발생률을 범위 기반으로 설정하였다.
    - 숙련 작업자 : +0~2%
    - 비숙련 작업자 : +0~4%

  - 오진열 발생 시  
    오진열 발생 상황에서는 오집 발생 가능성이 크게 증가한다고 가정하였다.
    - 숙련 작업자 : 오집 발생 확률 20%
    - 비숙련 작업자 : 오집 발생 확률 95%

- 시뮬레이션용 취소 사유 비율 설정  
  실제 운영 로직을 단순화하여 취소 사유는 아래 비율로 생성하였다.
  - 오집 : 60%
  - 미집 : 30%
  - 집품파손 : 5%
  - 리빈파손 : 5%

### 5-2. 재사용 규칙
- 집품파손, 리빈파손으로 분류된 상품은 재사용이 불가능하다.
- 오집 상품은 검수 통과 및 상품 상태 이상 없음 조건에서 재사용이 가능하다.
- 미집 상품은 상품 자체가 집품되지 않은 경우에 해당하므로 재사용과 직접 연결될 가능성이 낮다.
- 채소, 과일 카테고리는 품질 저하 가능성이 높아 다른 카테고리보다 재사용 가능 확률을 낮게 설정하였다.

## 6. 데이터 컬럼 설계

- 주문/상품 식별 영역(기본 키)  
  - `order_id` : 주문 단위 식별자  
  - `product_id` : 취소된 상품 식별자  
  - `category` : 상품 카테고리 (`채소`, `과일`, `가공식품`, `우유`, `축산`)  
  - `sku_option` : 동일 상품 내 옵션 정보

- 취소 발생 정보  
  - `cancel_reason` : 취소 사유 (`오집`, `미집`, `집품파손`, `리빈파손`)  
  - `damage_flag` : 파손 여부 (`집품파손`, `리빈파손`이면 1, 아니면 0)  
  - `quality_issue_flag` : 품질 저하 여부  
  - `stock_location` : 상품 보관 위치  
  - `cancel_time` : 취소 발생 시점  

- 작업 환경 정보  
  - `worker_type` : 작업자 숙련도 (`experienced`, `novice`)  
  - `is_peak_time` : 피크 시간대 여부(0/1)  
  - `location_known` : 상품 위치 정보 확인 가능 여부(0/1)

- 처리 속도 분석 영역  
  - `process_time` : 취소 처리 완료 시점  
  - `process_delay_min` : 취소 발생 후 처리 완료까지 걸린 시간(분)  

- 재고 재사용 분석 영역  
  - `restock_flag` : 취소 상품이 재고로 다시 반영되었는지 여부(0/1)  
  - `inspection_flag` : 검수 진행 및 통과 여부(0/1)  
  - `reused_flag` : 다른 주문에 재사용되었는지 여부(0/1)  
  - `additional_cancel_flag` : 재사용 실패 이후 추가 취소 발생 여부(0/1)

- 재사용 상품 탐색 시간 영역  
  - `search_start_time` : 동일 상품 재사용 판단 시작 시점  
  - `search_end_time` : 실제 상품 탐색 완료 시점  
  - `search_time_min` : 재사용 가능 상품 탐색 소요 시간(분)

## 7. 핵심지표
- 재사용률 : 재사용된 취소 재고 / 전체 취소 재고
- 처리 지연 시간 : 지연 시간에 따른 재사용 성공, 실패 비교
- 상품 탐색 시간 : 탐색 시간 감소율
- 추가 취소 발생률(재사용 실패율)
## 8. 분석흐름
- 처리 지연 시간과 재사용률 관계 분석
- 상품군별 오집/미집 발생률 분석
- 재사용 실패와 추가 취소 발생 관계 분석
- 과발생 구간 식별

## 9. 예상 인사이트 제시
- 처리 지연 시간이 지연되는 경우 재사용률 감소
- 특정 상품군에서 오집 발생 집중
- 재사용 실패 시 추가 취소 증가
- 일부 상품군은 재사용률 높음

## 10. 개선 방안
- 취소 전산 처리 자동화 도입
- 실시간 재고 반영
- 과발생 상품군 집중 관리

## 11. 핵심 메세지
- 취소 상품 전산처리 자동화를 통해 재사용 가능한 재고의 데이터를 생성하고, 재사용률 및 오집/미집 과발생 구간을 분석하여 추가 취소 감소를 위한 운영 개선 방안을 제시하였다.

## 12. 구현전 고려사항
(1) 시뮬레이션 규칙 (분석 가정)
- 취소 발생 비율 : 5~15%
- 취소 사유 비율 (전체 취소 기준)
  - 단순 주문 취소 : 70%
  - 운영 취소      : 30%
      └ 오집     : 18%  (운영취소의 60%)
      └ 미집     :  9%  (운영취소의 30%)
      └ 집품파손 :  1.5% (운영취소의 5%)
      └ 리빈파손 :  1.5% (운영취소의 5%)
- 처리 지연 시간 : 0~100분 균등 분포
- 탐색 시간 : 0~3분(재고 정보 여부에 따라 차등)

(2) 운영 기준 정의 (분석 기준)
- 처리 지연 30분 초과 시 재사용률 감소 가정
- 재사용 원 상품군 : 동일 SKU / 동일 옵션
- 탐색 시간 증가 요인 : 재고 위치 정보 미존재
- 재고 정리 상태에 따른 탐색 시간 차이 반영
